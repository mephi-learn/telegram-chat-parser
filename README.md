# Парсер чатов Telegram

Этот проект представляет собой клиент-серверное приложение для парсинга экспортированных чатов Telegram и обогащения данных об участниках через Telegram API.

## Возможности

* Парсинг файлов экспорта чатов Telegram (JSON).
* Извлечение участников (авторов и упоминаний).
* Обогащение данных об участниках (имя, имя пользователя, био) через Telegram API.
* Клиент-серверная архитектура.
* Асинхронная обработка задач с получением `task_id`.
* Кэширование результатов на основе хеша файла.
* Поддержка получения результатов по хешу файла.
* Пагинация для больших наборов результатов.
* Адаптивное ограничение частоты запросов к Telegram API.
* Управление сервером как фоновым процессом (демоном).
* Эндпоинт для проверки состояния (`/health`).

## Архитектура

Приложение состоит из двух основных компонентов:

* **Сервер:** Отвечает за обработку файлов чатов, взаимодействие с Telegram API, а также управление задачами и результатами.
* **Клиент:** CLI-инструмент для отправки файлов на сервер и получения результатов.

## Требования

* Go 1.21+
* Учетные данные Telegram API (API ID, API Hash, номер телефона)

## Конфигурация

Конфигурация приложения может быть задана через файл `config.yml` или переменные окружения.

### Приоритет источников

Параметры загружаются в следующем порядке (каждый последующий источник может переопределить предыдущий):
1.  **Файл `config.yml`**: базовые настройки.
2.  **Файл `.env`**: переменные окружения из файла (удобно для локальной разработки).
3.  **Переменные окружения системы**: наивысший приоритет, идеально для Docker и CI/CD.

### Параметры конфигурации

| YAML-путь | Переменная окружения | Описание | По умолчанию |
| :--- | :--- | :--- | :--- |
| `server.host` | `SERVER_HOST` | Хост, на котором запускается сервер. | `"0.0.0.0"` |
| `server.port` | `SERVER_PORT` | Порт сервера. | `8080` |
| `telegram_api.api_id` | `API_ID` | **(Обязательно)** Ваш API ID от Telegram. | - |
| `telegram_api.api_hash` | `API_HASH` | **(Обязательно)** Ваш API Hash от Telegram. | - |
| `telegram_api.phone_number` | `PHONE_NUMBER` | **(Обязательно)** Ваш номер телефона в международном формате. | - |
| `telegram_api.session_file` | `SESSION_FILE` | Файл для хранения сессии Telegram. | `"tg.session"` |
| `processing.task_timeout_seconds`| `TASK_TIMEOUT_SECONDS` | Таймаут на обработку одной задачи в секундах (0 - без таймаута). | `30` |
| `processing.cache_ttl_minutes` | `CACHE_TTL_MINUTES` | Время жизни (TTL) для кэшированных результатов в минутах. | `60` |
| `rate_limiter.strategy` | `RATE_LIMITER_STRATEGY`| Стратегия ограничения запросов: `always_delay` или `adaptive`. | `always_delay`|
| `rate_limiter.request_delay_ms` | `REQUEST_DELAY` | Задержка между запросами к Telegram API в миллисекундах. | `1000` |
| `rate_limiter.throttled_retries` | `THROTTLED_RETRIES` | Количество повторных попыток при получении ошибки `FLOOD_WAIT`. | `5` |
| `rate_limiter.cooldown_duration_seconds` | `COOLDOWN_DURATION_SECONDS` | Длительность "остывания" после троттлинга в секундах. | `60` |
| `service.pid_file_path` | `PID_FILE_PATH` | Путь к PID-файлу при запуске в режиме демона. | `telegram-chat-parser-server.pid` |


## Запуск

### Сервер

Сервер поддерживает два режима запуска: интерактивный (для отладки) и фоновый (для продакшена).

#### Интерактивный режим

Запускает сервер в текущей консоли. Все логи выводятся в `stdout`. Остановка по `Ctrl+C`.

```bash
# Убедитесь, что config.yml или переменные окружения настроены
./bin/server -action=run
# или просто
./bin/server
```

#### Фоновый режим (демон)

Управляет сервером как системным сервисом.

```bash
# Запустить сервер в фоновом режиме
./bin/server -action=start

# Проверить статус (через поиск процесса)
ps aux | grep "server -action=run"

# Остановить сервер
./bin/server -action=stop

# Перезапустить сервер
./bin/server -action=restart
```

### Клиент

Клиент используется для отправки файлов на сервер. Он также использует `config.yml` для определения адреса сервера (`server.host` и `server.port`).

```bash
# Обработать файл и получить task_id
./bin/client -file /path/to/chat.json

# Обработать по хешу (если результат уже есть в кэше сервера)
./bin/client -hash <sha256_of_file_content>
```
